After completing an exam, EGEL Simulator automatically saves your results to a persistent history log. The HistoryPage provides comprehensive insights into your performance, allowing you to track progress and identify areas for improvement.

## Accessing Your History

There are two ways to access the HistoryPage:

<Steps>
  <Step title="Automatic Redirect After Exam">
    When you complete an exam by clicking **"Finalizar"** on the last question, the application:
    
    1. Calculates your final score
    2. Creates a history entry
    3. Automatically navigates to `/history`
    
    ```typescript
    // From TestPage.tsx:73
    navigate("/history");
    ```
  </Step>

  <Step title="Manual Navigation from Home">
    You can also access your history at any time by:
    
    - Navigating to the home page
    - Clicking the history or results button
    - Being routed to `/history`
    
    From the HistoryPage, click **"← Volver"** to return home:
    
    ```typescript
    // Back button from HistoryPage.tsx:28-30
    <button onClick={() => navigate('/home')}>
        <ArrowLeft className="h-5 w-5 mr-1" /> Volver
    ```

</Steps>

## Understanding Score Calculations

Scores are calculated when you complete an exam:

### Score Computation

```typescript
// From TestPage.tsx:53-57
const total = examQuestions.length
const correct = Math.floor(total * 0.7)  // Example: placeholder logic
const score = Math.round((correct / total) * 100)
```

<Note>
The current implementation uses a placeholder calculation (70% correct). In a production environment, this would compare your selected answers against `question.correctAnswerIndex` for each question.
</Note>

### Real Score Calculation Example

For actual score tracking, the logic would be:

```typescript
// Pseudo-code for real scoring
const correct = examQuestions.reduce((count, question) => {
    const userAnswer = selectedAnswers[question.id];
    const isCorrect = userAnswer === question.correctAnswerIndex;
    return count + (isCorrect ? 1 : 0);
}, 0);

const score = Math.round((correct / total) * 100);
```

### Score Display

Scores are shown as:
- **Percentage**: 0-100%
- **Correct count**: "X aciertos de Y"
- **Large, bold text** in blue for emphasis

## History Entry Creation

Entries are created automatically when you finish an exam:

```typescript
// Entry creation from TestPage.tsx:58-71
const now = new Date();
const date = now.toLocaleDateString()
const time = now.toLocaleTimeString()

addHistoryEntry({
    date,
    time,
    score,
    correct,
    total,
    type: config!.area,
    timerEnabled: config!.timerEnabled,
    practiceMode: config!.practiceMode,
});
```

The `addHistoryEntry` function:
1. Generates a unique UUID for the entry
2. Prepends the new entry to the history array (newest first)
3. Persists the updated history to localStorage

```typescript
// From useHistoryStore.tsx:26-31
addEntry: (entry) => {
    const newEntry: HistoryItem = {
        id: crypto.randomUUID(),
        ...entry,
    };
    set({ history: [newEntry, ...get().history] });
}
```

## Persistent Storage

History is saved using Zustand's persist middleware:

```typescript
// Store definition from useHistoryStore.tsx:22-38
export const useHistoryStore = create<HistoryState>()()
    persist(
        (set, get) => ({
            history: [],
            addEntry: (entry) => {
                const newEntry: HistoryItem = {
                    id: crypto.randomUUID(),
                    ...entry,
                };
                set({ history: [newEntry, ...get().history] });
            },
            clearHistory: () => set({ history: [] }),
        }),
        {
            name: 'sim-history-store',  // localStorage key
        }
    )
);
```

This means:
- History persists across application restarts
- Data is stored in localStorage as `sim-history-store`
- No external database required for history
- Works offline

<Tip>
Your exam history is stored locally on your device. If you reinstall the application or clear browser data, your history will be lost. Back up important data if needed.
</Tip>

## Clearing History

While not exposed in the UI, the store provides a clear function:

```typescript
// From useHistoryStore.tsx:33
clearHistory: () => set({ history: [] })
```

This could be called programmatically or added to the UI:

```typescript
// Example usage
const { clearHistory } = useHistoryStore();

// Clear all history
clearHistory();
```

<Warning>
Clearing history is permanent and cannot be undone. All exam records will be permanently deleted from localStorage.
</Warning>

## Technical Implementation Notes

### History Rendering

Entries are rendered in reverse chronological order (newest first):

```typescript

{history.map((item) => (
    <div key={item.id}>
        {/* Entry card */}
    </div>
))}
```

The `map` iterates over the history array, which has new entries prepended:

```typescript
// From useHistoryStore.tsx:31
set({ history: [newEntry, ...get().history] });
```

### Badge Icons

Configuration badges use Lucide icons:

- **PlayCircle**: Indicates practice vs simulation mode
- **Timer**: Shows whether timer was enabled
- **CheckCircle**: Marks completed exams

```typescript
// Icons from HistoryPage.tsx:1
import { ClipboardList, ArrowLeft, CheckCircle, Timer, PlayCircle } from "lucide-react";
```

### Persistent State Sync

Zustand's persist middleware automatically:
- Saves to localStorage on state changes
- Loads from localStorage on app initialization
- Handles serialization/deserialization
- Maintains consistency across sessions

No manual save/load logic required!
