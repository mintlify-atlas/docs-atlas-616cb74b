## Practice Mode vs Simulation Mode

### Practice Mode

<Note>
Practice mode provides **immediate feedback** after each answer, showing you the correct answer before moving to the next question.
</Note>

When practice mode is **enabled**:
- You select an answer
- Feedback appears immediately showing the correct answer
- You can learn from mistakes in real-time
- Ideal for studying and reviewing material

```typescript
// From SetupPage.tsx:13
const [practiceMode, setPracticeMode] = useState(false);

// Toggle practice mode
<input
    id="practice-mode"
    type="checkbox"
    checked={practiceMode}
    onChange={() => setPracticeMode(!practiceMode)}
    className="form-checkbox w-5 h-5 text-green-600"
/>
```

Practice mode displays feedback through the Question component:

```typescript
// From Question.tsx:59-66
{showFeedback && mode === "practica" && (
    <div className="mt-4 p-4 bg-blue-50 border border-blue-200 rounded-xl text-blue-800 text-sm">
        <strong>Respuesta correcta:</strong>{" "}
        <span className="font-medium">
            {String.fromCharCode(65 + question.correctAnswerIndex)}. {question.options[question.correctAnswerIndex]}
        </span>
    </div>
)}
```

### Simulation Mode

When practice mode is **disabled** (simulation mode):
- You can select and change answers freely
- No feedback is provided during the exam
- Results are shown only at the end via the HistoryPage
- Mimics the actual EGEL exam experience

<Warning>
In simulation mode, you won't see the correct answers until the exam is complete. This tests your knowledge under exam-like conditions.
</Warning>

## Timer Configuration

### Enabling/Disabling the Timer

The timer is **enabled by default** but can be toggled:

```typescript
// From SetupPage.tsx:12
const [timerEnabled, setTimerEnabled] = useState(true);

// Timer checkbox from SetupPage.tsx:89-96
<input
    id="enable-timer"
    type="checkbox"
    checked={timerEnabled}
    onChange={() => setTimerEnabled(!timerEnabled)}
    className="form-checkbox w-4 h-4 text-green-600"
/>
```

<Tip>
The timer checkbox is positioned in the bottom-right corner of the setup interface, separate from other configuration options.
</Tip>

### Timer Duration

The exam duration is currently **hardcoded to 40,000 seconds** (approximately 11 hours):

```typescript
// From SetupPage.tsx:16-21
const handleClick = () => {
    const simulationSettings = {
        area: selectedArea,
        timerEnabled: timerEnabled,
        duration: 40000,  // Fixed duration in seconds
        practiceMode: practiceMode,
    };
    setConfig(simulationSettings);
};
```

<Note>
While 40,000 seconds seems long, this value can be adjusted in the source code if you need a different duration for your practice sessions.
</Note>

### How the Timer Works During Exams

Once the exam starts, the timer counts down from the configured duration:

```typescript
// Timer implementation from TestPage.tsx:29-39
useEffect(() => {
    if (!config?.timerEnabled) return
    const interval = setInterval(() => {
        setTimeLeft((prev) => {
            if (prev <= 1) clearInterval(interval)
            return prev - 1
        })
    }, 1000)
    return () => clearInterval(interval)
}, [config])
```

The timer displays:
- In `MM:SS` format
- With a clock icon
- Turns red and pulses when less than 5 minutes remain (300 seconds)

```typescript
// Time formatting from TestPage.tsx:80-84
const formatTime = (seconds: number) => {
    const min = Math.floor(seconds / 60)
    const sec = seconds % 60
    return `${min}:${sec.toString().padStart(2, "0")}`
}
```

## Configuration State Management

All configuration is stored in Zustand's `useSetupStore`:

```typescript
// Store definition from useSetupStore.tsx:3-14
type SimulationSettings = {
  area: SimulationType;
  timerEnabled: boolean;
  duration: number;
  practiceMode: boolean;
};

type SetupStore = {
  config: SimulationSettings | null;
  setConfig: (config: SimulationSettings) => void;
  resetConfig: () => void;
};
```

When you click **"Iniciar simulacro"**, the configuration is saved:

```typescript
// From SetupPage.tsx:16-25
const handleClick = () => {
    const simulationSettings = {
        area: selectedArea,
        timerEnabled: timerEnabled,
        duration: 40000,
        practiceMode: practiceMode,
    };
    setConfig(simulationSettings);  // Store configuration
    
    navigate("/test");  // Navigate to exam
};
```

## Configuration Validation

The TestPage validates configuration on load:

```typescript
// Configuration check from TestPage.tsx:23-27
useEffect(() => {
    if (!config) {
        navigate("/setup");  // Redirect if no config
    }
}, [config, navigate])
```

If you navigate directly to `/test` without configuring:
- You'll be automatically redirected to `/setup`
- Must complete configuration before starting
- Prevents undefined behavior during exam

<Warning>
Direct navigation to the test page is blocked without proper configuration. Always start from the SetupPage.
</Warning>

## Best Practices

1. **Start with Practice Mode** when learning new material
2. **Use Simulation Mode** for final exam preparation
3. **Enable Timer** to build time management skills
4. **Practice Both Areas** to prepare comprehensively
5. **Focus Study** on one area when reviewing specific topics
6. **Disable Timer** when you want to thoroughly understand concepts
7. **Track Progress** by reviewing your history after each session
